# 上架下架功能说明

## 功能概述
为菜品管理添加了上架/下架功能，管理员可以控制菜品在用户端的显示状态。

## 实现细节

### 1. 数据库字段
- 使用 `status` 字段标识菜品状态
  - `1`: 已上架（用户端可见）
  - `0`: 已下架（用户端不可见）

### 2. 管理端功能
**菜品管理页面** (`pages/admin/admin.vue`)
- 显示所有菜品（包括已上架和已下架）
- 每个菜品显示当前状态标签：
  - 绿色"已上架"
  - 红色"已下架"
- 操作按钮：
  - 已上架菜品：显示"下架"按钮（橙色）
  - 已下架菜品：显示"上架"按钮（绿色）
  - 编辑按钮（蓝色）
  - 删除按钮（红色）

### 3. 用户端功能
**首页** (`pages/index/index.vue`)
- 只显示已上架的菜品（`status === 1`）
- 已下架的菜品自动隐藏，用户无法看到和购买

### 4. API 修改
**云函数** (`cloudfunctions/dish/index.js`)
- `getAll` 方法支持 `includeInactive` 参数
  - `true`: 返回所有菜品（管理端使用）
  - `false`: 只返回已上架菜品（用户端使用）

**前端 API** (`utils/api.js`)
- `dishes.getAll(includeInactive)` 方法
  - 管理端调用：`api.dishes.getAll(true)`
  - 用户端调用：`api.dishes.getAll(false)`

## 使用流程

### 管理员操作
1. 登录管理员账号（admin / admin123）
2. 进入"菜品管理"
3. 找到需要下架的菜品，点击"下架"按钮
4. 确认操作后，菜品状态变为"已下架"
5. 用户端立即无法看到该菜品

### 重新上架
1. 在菜品管理中找到已下架的菜品
2. 点击"上架"按钮
3. 确认操作后，菜品状态变为"已上架"
4. 用户端立即可以看到该菜品

## 应用场景

1. **季节性菜品**：根据季节上架/下架特定菜品
2. **临时缺货**：食材不足时临时下架，补货后重新上架
3. **新菜品测试**：先下架状态创建，准备好后再上架
4. **特殊活动**：活动期间上架特定菜品，活动结束后下架

## 注意事项

1. **已下架菜品不会被删除**，只是在用户端隐藏
2. **购物车中的菜品**：如果用户已将菜品加入购物车，下架后购物车中仍保留（建议后续优化）
3. **订单中的菜品**：历史订单中的菜品不受影响
4. **默认状态**：新创建的菜品默认为已上架状态（`status: 1`）

## 部署步骤

1. 重新上传 `dish` 云函数
2. 在微信开发者工具中右键 `cloudfunctions/dish`
3. 选择"上传并部署：云端安装依赖"
4. 重新编译前端项目

## 数据库更新

如果现有菜品没有 `status` 字段，需要在云开发控制台手动更新：
1. 打开云开发控制台
2. 进入数据库 → dishes 集合
3. 为所有现有菜品添加 `status: 1` 字段
