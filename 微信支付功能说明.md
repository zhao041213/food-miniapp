## 实现流程

### 1. 提交订单
**购物车页面** (`pages/cart/cart.vue`)
- 用户点击"提交订单"
- 创建订单，初始状态为"待支付"
- 自动调用微信支付接口

### 2. 统一下单
**订单云函数** (`cloudfunctions/order/index.js`)
- 新增 `unifiedOrder` action
- 调用微信支付统一下单API
- 参数包括：
  - 订单号（out_trade_no）
  - 金额（total_fee，单位：分）
  - 用户openid
  - 商品描述："微信点餐-订单支付"
- 生成支付签名并返回支付参数

### 3. 调起支付
**前端调用** (`pages/cart/cart.vue`, `pages/order/order.vue`)
- 使用 `wx.requestPayment()` 调起微信支付
- 传入支付参数：
  - timeStamp: 时间戳
  - nonceStr: 随机字符串
  - package: 预支付交易会话标识
  - signType: 签名方式（MD5）
  - paySign: 支付签名

### 4. 支付结果处理
- **支付成功**：
  - 更新订单状态为"制作中"
  - 清空购物车
  - 跳转到订单页面
- **支付失败/取消**：
  - 订单保持"待支付"状态
  - 用户可在订单页面重新支付

## 订单状态流转

```
待支付 → 制作中 → 待上菜 → 已上齐
```

- **待支付**: 订单创建后的初始状态
- **制作中**: 支付成功后自动更新
- **待上菜**: 管理员在后台更新
- **已上齐**: 用户或管理员标记完成

### 签名算法
1. 将所有参数按字典序排序
2. 拼接成 `key1=value1&key2=value2` 格式
3. 末尾追加 `&key=商户密钥`
4. 进行MD5加密并转大写

### API接口
- **统一下单**: `https://api.mch.weixin.qq.com/pay/unifiedorder`
- **请求格式**: XML
- **响应格式**: XML

## 注意事项

### 1. 支付回调地址
当前代码中的回调地址为占位符：
```javascript
notify_url: 'https://your-domain.com/pay/notify'
```

**生产环境需要配置真实的回调地址**：
- 必须是公网可访问的HTTPS地址
- 用于接收微信支付结果通知
- 需要验证签名并更新订单状态

### 2. 金额单位
- 前端显示：元（小数）
- 微信支付：分（整数）
- 转换公式：`Math.round(totalPrice * 100)`

### 3. 用户登录
- 必须使用微信授权登录
- 需要获取用户的openid
- 手机号登录的用户无法使用微信支付

### 4. 测试环境
- 开发阶段可以使用微信支付沙箱环境
- 需要在商户平台开通沙箱功能
- 沙箱环境不会真实扣款

## 安全建议

1. **密钥保护**
   - 商户密钥不要提交到代码仓库
   - 建议使用环境变量或云函数配置

2. **签名验证**
   - 支付回调必须验证签名
   - 防止恶意请求篡改订单

3. **金额校验**
   - 统一下单前验证订单金额
   - 防止前端篡改价格

4. **订单幂等**
   - 使用订单ID作为商户订单号
   - 防止重复支付

## 常见问题

### Q1: 支付时提示"商户号配置错误"
- 检查商户号和密钥是否正确
- 确认小程序已在商户平台关联

### Q2: 支付成功但订单状态未更新
- 检查支付回调地址是否配置
- 查看云函数日志排查错误

### Q3: 调起支付失败
- 确认用户已微信授权登录
- 检查openid是否正确获取
- 查看统一下单返回的错误信息
