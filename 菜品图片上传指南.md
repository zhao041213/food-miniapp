# 菜品图片上传完整指南

## 快速开始（更新现有菜品图片）

### 第一步：上传云函数

1. 在微信开发者工具中，右键 `cloudfunctions/uploadDishes` 文件夹
2. 选择"上传并部署：云端安装依赖"
3. 等待上传完成

### 第二步：运行云函数更新图片

**方法A：在云开发控制台运行**

1. 点击微信开发者工具顶部"云开发"按钮
2. 进入"云函数"页面
3. 找到 `uploadDishes` 云函数，点击进入
4. 点击"测试"标签
5. 在测试参数中输入：
```json
{
  "action": "updateWithPlaceholder"
}
```
6. 点击"运行测试"
7. 查看返回结果，应该显示"更新成功"和更新的菜品数量

**方法B：在小程序中调用**

在微信开发者工具的控制台中运行：
```javascript
wx.cloud.callFunction({
  name: 'uploadDishes',
  data: {
    action: 'updateWithPlaceholder'
  },
  success: res => {
    console.log('更新成功:', res)
  },
  fail: err => {
    console.error('更新失败:', err)
  }
})
```

### 第三步：验证数据

1. 在云开发控制台 → "数据库"
2. 点击 `dishes` 集合
3. 查看每条菜品数据的 `image` 字段
4. 应该都已经更新为占位图URL

### 第四步：在小程序中查看

1. 刷新小程序
2. 进入首页，应该能看到带图片的菜品列表
3. 图片是彩色占位图，显示菜品名称

## 进阶：上传真实图片

### 方式1：手动上传到云存储

1. **准备图片**
   - 准备菜品图片（建议尺寸：400x300 或 800x600）
   - 格式：JPG 或 PNG

2. **上传到云存储**
   - 云开发控制台 → "云存储"
   - 点击"上传文件"
   - 创建文件夹 `dishes/`
   - 上传图片

3. **获取图片URL**
   - 点击上传的图片
   - 复制"文件ID"（类似：`cloud://xxx.xxx/dishes/dish-1.jpg`）

4. **更新数据库**
   - 在数据库中找到对应的菜品
   - 将 `image` 字段更新为云存储的文件ID

### 方式2：使用云函数批量更新

如果你已经上传了图片到云存储，可以调用云函数批量更新：

```javascript
wx.cloud.callFunction({
  name: 'uploadDishes',
  data: {
    action: 'updateImage',
    data: {
      dishId: '菜品ID',
      fileID: 'cloud://xxx.xxx/dishes/dish-1.jpg'
    }
  }
})
```

## 常见问题

### Q1: 占位图显示不出来
- 检查网络连接
- 占位图服务可能被墙，可以直接上传真实图片到云存储

### Q2: 如何批量更新图片
- 使用云函数的 `updateImage` action
- 或者在数据库中直接编辑

### Q3: 图片太大加载慢
- 压缩图片，建议单张不超过200KB
- 使用云存储的CDN加速
