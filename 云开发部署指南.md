## 接下来的操作步骤

### 第一步：在云开发控制台创建数据库集合

1. **打开微信开发者工具**
2. **点击顶部"云开发"按钮**
3. **进入"数据库"标签**
4. **创建以下集合**（点击"+"号）：

   - `users` - 用户表
   - `dishes` - 菜品表
   - `orders` - 订单表

### 第三步：上传并部署云函数

1. **在 HBuilderX 中**，找到项目目录下的 `cloudfunctions` 文件夹

2. **右键点击 `user` 文件夹**
   - 选择 **"上传并部署：云端安装依赖"**
   - 等待上传完成

3. **右键点击 `dish` 文件夹**
   - 选择 **"上传并部署：云端安装依赖"**

4. **右键点击 `order` 文件夹**
   - 选择 **"上传并部署：云端安装依赖"**
   - 等待上传完成

### 第四步：测试云函数

在微信开发者工具的云开发控制台：

1. 进入 **"云函数"** 标签
2. 点击某个云函数（如 `user`）
3. 点击 **"测试"** 按钮
4. 输入测试参数：
   ```json
   {
     "action": "login",
     "data": {
       "phone": "admin",
       "password": "admin123"
     }
   }
   ```
5. 点击 **"运行测试"**，查看返回结果

### 第五步：运行小程序

1. **在 HBuilderX 中重新编译**

## 常见问题

### Q1: 云函数上传失败？
- 检查网络连接
- 确认已开通云开发
- 重启 HBuilderX 和微信开发者工具

### Q2: 调用云函数报错？
- 检查云函数是否已上传
- 查看云开发控制台的云函数日志
- 确认环境 ID 是否正确

### Q3: 数据库操作失败？
- 检查数据库集合是否已创建
- 查看云函数日志中的错误信息
- 确认数据格式是否正确

### Q4: 前端无法调用云函数？
- 检查 App.vue 中的云开发初始化代码
- 确认环境 ID 是否正确
- 查看控制台错误信息

## 云开发 vs 传统后端对比

| 项目 | 云开发 | 传统后端 |
|------|--------|----------|
| 服务器 | 无需购买 | 需要购买 |
| 域名 | 无需配置 | 需要购买和备案 |
| HTTPS | 自动配置 | 需要配置证书 |
| 维护 | 微信官方维护 | 自己维护 |
| 费用 | 有免费额度 | 服务器费用 |
| 扩展性 | 自动扩展 | 需要手动扩展 |
