# 微信点餐小程序

基于 uni-app 开发的微信点餐小程序，包含用户端和管理端功能。后端使用 Node.js + Express + MySQL 实现数据持久化。

## 功能特性

### 用户端
- 📱 首页菜品展示，支持分类筛选
- 🔍 菜品搜索功能
- 🛒 购物车功能，可增减菜品数量
- 💰 订单结算（支付接口暂未实现）
- 📋 订单列表，查看订单状态
- 👤 用户登录/退出

### 管理端
- ⚙️ 菜品管理（增删改查）
- 📷 图片上传功能
- 📊 管理员权限控制

## 技术栈

### 前端
- uni-app
- Vue.js
- 微信小程序

### 后端
- Node.js
- Express
- MySQL

## 快速开始

### 环境要求

**前端：**
- HBuilderX（最新版）
- 微信开发者工具

**后端：**
- Node.js 14+
- MySQL 5.7+ 或 8.0+
- npm 或 yarn

### 安装步骤

#### 第一步：启动后端服务

1. **安装依赖**
```bash
cd server
npm install
```

2. **配置数据库**

复制环境变量配置文件：

**Windows 用户：**
- 方法一：在 `server` 文件夹中，复制 `.env.example` 文件，重命名为 `.env`
- 方法二：使用 PowerShell 命令
  ```powershell
  Copy-Item .env.example .env
  ```

**Mac/Linux 用户：**
```bash
cp .env.example .env
```

编辑 `.env` 文件，修改数据库配置：
```env
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=你的MySQL密码
DB_NAME=order_system

PORT=3001
```

3. **创建数据库和表**

方式一：使用命令行
```bash
mysql -u root -p < database.sql
```

方式二：使用 MySQL 客户端
- 打开 MySQL Workbench 或 Navicat
- 连接到数据库
- 执行 `server/database.sql` 文件中的 SQL 语句

4. **启动后端服务**
```bash
# 开发模式（代码修改自动重启）
npm run dev

# 生产模式
npm start
```

看到以下信息表示启动成功：
```
数据库连接成功
服务器运行在 http://localhost:3001
```

5. **测试后端接口**

浏览器访问：`http://localhost:3001/api/health`

应该返回：
```json
{
  "status": "ok",
  "message": "服务运行正常"
}
```

#### 第二步：配置并运行前端

1. **用 HBuilderX 打开项目**
   - 打开 HBuilderX
   - 文件 → 打开目录 → 选择项目根目录 `D:\微信点餐小程序`

2. **配置 API 地址**（可选）

如果后端不是运行在 `localhost:3001`，需要修改：
   - 打开 `utils/api.js`
   - 修改第 2 行：
   ```javascript
   const API_BASE_URL = 'http://你的后端地址:端口/api'
   ```

3. **配置微信开发者工具**
   - 打开微信开发者工具
   - 设置 → 安全设置 → 服务端口：开启
   - 详情 → 本地设置 → 勾选"不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书"

4. **补充图片资源**（可选，不影响功能）

需要手动添加以下图片：
   - TabBar 图标（`static/tabbar/`）：
     - `home.png` / `home-active.png`（首页图标）
     - `order.png` / `order-active.png`（订单图标）
     - `mine.png` / `mine-active.png`（我的图标）
   - 菜品占位图：`static/dishes/placeholder.png`
   - 用户头像：`static/avatar.png`

5. **运行小程序**
   - 在 HBuilderX 中点击"运行" → "运行到小程序模拟器" → "微信开发者工具"
   - 首次运行会自动打开微信开发者工具并加载项目
   - 如果提示 AppID 错误，在微信开发者工具中选择"测试号"

#### 第三步：测试功能

1. **测试用户端**
   - 浏览首页菜品（数据来自数据库）
   - 添加菜品到购物车
   - 提交订单
   - 查看订单列表

2. **测试管理端**
   - 点击"我的"标签
   - 使用管理员账号登录：`admin` / `admin123`

### 常见问题

**Q1: 后端启动失败，提示数据库连接错误？**
- 检查 MySQL 服务是否启动
- 检查 `.env` 文件中的数据库配置是否正确
- 检查数据库用户名和密码是否正确

**Q2: 前端无法获取数据？**
- 检查后端服务是否正常运行
- 检查 `utils/api.js` 中的 API 地址是否正确（默认：`http://localhost:3001/api`）
- 检查微信开发者工具是否关闭了域名校验
- 打开控制台查看具体错误信息

**Q3: 小程序提示 request:fail？**
- 确保后端服务已启动
- 在微信开发者工具中关闭域名校验（详情 → 本地设置）
- 检查网络连接

**Q4: 管理员无法登录？**
- 确保已执行 `database.sql` 创建了默认账号
- 检查数据库中 users 表是否有数据
- 尝试使用 SQL 查询：`SELECT * FROM users WHERE role='admin'`

## 功能说明

### 用户端流程

1. **浏览菜品**
   - 打开小程序，首页展示所有菜品
   - 点击顶部分类切换（全部、热菜、凉菜、主食、汤类）
   - 菜品数据实时从数据库获取

2. **加入购物车**
   - 点击菜品右下角的"+"按钮
   - 购物车图标显示商品数量和总价
   - 购物车数据暂存在本地

3. **结算下单**
   - 点击底部购物车按钮进入购物车页面
   - 可增减菜品数量或删除菜品
   - 点击"提交订单"按钮
   - 订单提交到数据库保存

4. **查看订单**
   - 点击底部"订单"标签
   - 查看所有订单及状态
   - 可将订单标记为"已上齐"
   - 订单状态实时同步到数据库

5. **用户登录**
   - 点击底部"我的"标签
   - 点击"点击登录"
   - 输入账号密码登录
   - 登录信息验证数据库

### 管理端流程

1. **管理员登录**
   - 进入"我的"页面
   - 使用管理员账号登录：`admin` / `admin123`
   - 登录后显示"菜品管理"入口

2. **菜品管理**
   - 点击"菜品管理"进入管理页面
   - 查看所有菜品列表（从数据库读取）

3. **添加菜品**
   - 点击"+ 添加菜品"按钮
   - 填写菜品信息（名称、价格、分类、描述）
   - 点击"保存"，数据写入数据库
   - 用户端立即可见新菜品

4. **编辑菜品**
   - 点击菜品的"编辑"按钮
   - 修改菜品信息
   - 点击"保存"，更新数据库
   - 用户端刷新后看到最新信息

5. **删除菜品**
   - 点击菜品的"删除"按钮
   - 确认删除
   - 从数据库中删除
   - 用户端不再显示该菜品

### 数据同步机制

- **管理端 → 数据库 → 用户端**
  - 管理员修改菜品 → 立即保存到 MySQL
  - 用户刷新页面 → 从数据库获取最新数据
  - 实现前后端数据实时同步

- **用户端 → 数据库**
  - 用户提交订单 → 保存到数据库
  - 更新订单状态 → 同步到数据库
  - 所有订单数据持久化存储

## 数据存储架构

### 数据库存储（MySQL）
- **菜品数据**：`dishes` 表
  - 管理员增删改查的菜品信息
  - 用户端展示的菜品列表
  
- **订单数据**：`orders` 和 `order_items` 表
  - 用户提交的订单主信息
  - 订单中的菜品详情
  
- **用户数据**：`users` 表
  - 用户账号、密码、角色
  - 登录验证信息

### 本地存储（localStorage）
- **购物车数据**：`cart`
  - 临时存储用户选择的菜品
  - 提交订单后清空
  
- **用户登录状态**：`userInfo`
  - 保存登录用户信息
  - 用于权限判断

### 数据流向

```
用户端 ←→ API 接口 ←→ 数据库
  ↓
购物车（本地）
  ↓
提交订单 → 数据库
```

## API 接口文档

详细文档见 `server/README.md`

### 菜品接口
| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| GET | `/api/dishes` | 获取菜品列表 | 所有用户 |
| GET | `/api/dishes/:id` | 获取单个菜品 | 所有用户 |
| POST | `/api/dishes` | 添加菜品 | 管理员 |
| PUT | `/api/dishes/:id` | 更新菜品 | 管理员 |
| DELETE | `/api/dishes/:id` | 删除菜品 | 管理员 |

### 订单接口
| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| GET | `/api/orders` | 获取订单列表 | 所有用户 |
| GET | `/api/orders/:id` | 获取单个订单 | 所有用户 |
| POST | `/api/orders` | 创建订单 | 所有用户 |
| PUT | `/api/orders/:id/status` | 更新订单状态 | 所有用户 |
| DELETE | `/api/orders/:id` | 删除订单 | 管理员 |

### 用户接口
| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| POST | `/api/users/login` | 用户登录 | 公开 |
| POST | `/api/users/register` | 用户注册 | 公开 |
| GET | `/api/users/:id` | 获取用户信息 | 登录用户 |

### 接口调用示例

**获取菜品列表：**
```javascript
// 前端调用
import api from '@/utils/api.js'

const res = await api.dishes.getAll()
console.log(res.data) // 菜品数组
```

**创建订单：**
```javascript
const orderData = {
  userId: 1,
  items: [
    { id: 1, name: '宫保鸡丁', price: 28, count: 2 }
  ],
  totalPrice: 56
}

const res = await api.orders.create(orderData)
console.log(res.data) // 订单信息
```

## 开发注意事项

### 前端开发
1. **域名校验**
   - 开发时必须在微信开发者工具中关闭域名校验
   - 生产环境需要配置合法域名并启用 HTTPS

2. **API 地址配置**
   - 本地开发：`http://localhost:3001/api`
   - 生产环境：修改 `utils/api.js` 中的 `API_BASE_URL`

3. **图片资源**
   - TabBar 图标建议尺寸：81px × 81px
   - 菜品图片建议尺寸：400px × 400px
   - 支持 PNG、JPG 格式

4. **数据刷新**
   - 页面使用 `onShow` 生命周期刷新数据
   - 确保从后端获取最新数据

### 后端开发
1. **数据库连接**
   - 使用连接池管理数据库连接
   - 默认最大连接数：10

2. **错误处理**
   - 所有接口都有统一的错误处理
   - 返回格式：`{ success: false, message: '错误信息' }`

3. **CORS 配置**
   - 已启用 CORS，允许跨域请求
   - 生产环境建议配置白名单

4. **安全性**
   - 密码未加密（建议使用 bcrypt）
   - 未实现 JWT 认证（建议添加）
   - 未实现接口权限验证（建议添加）

### 部署注意事项

**前端部署：**
- 小程序需要在微信公众平台配置服务器域名
- 域名必须支持 HTTPS
- 需要上传代码审核后才能发布

**后端部署：**
- 建议使用 PM2 管理 Node.js 进程
- 配置 Nginx 反向代理
- 启用 HTTPS（使用 Let's Encrypt 免费证书）
- 定期备份数据库

**数据库部署：**
- 生产环境建议使用云数据库（如阿里云 RDS）
- 定期备份数据
- 配置主从复制提高可用性
