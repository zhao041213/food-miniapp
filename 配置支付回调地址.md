# 配置云开发HTTP触发器作为支付回调地址

## 步骤1：上传支付回调云函数

1. 在微信开发者工具中，找到 `cloudfunctions/payCallback` 文件夹
2. 右键点击该文件夹
3. 选择"上传并部署：云端安装依赖"
4. 等待部署完成

## 步骤2：获取HTTP触发器地址

1. 部署完成后，右键 `cloudfunctions/payCallback` 文件夹
2. 选择"云函数详情"
3. 在弹出的页面中找到"触发器"标签
4. 复制HTTP触发器的URL地址（格式类似：`https://xxx.service.tcloudbase.com/payCallback`）

**或者通过云开发控制台获取：**
1. 打开微信开发者工具的"云开发"控制台
2. 进入"云函数" → 找到 `payCallback`
3. 点击进入详情页
4. 在"触发器"标签中查看HTTP触发路径
5. 完整地址格式：`https://你的环境ID.service.tcloudbase.com/payCallback`

## 步骤3：更新order云函数中的回调地址

获取到HTTP触发器地址后，需要更新 `cloudfunctions/order/index.js` 中的 `notify_url`：

```javascript
// 找到这一行
notify_url: 'https://your-domain.com/pay/notify',

// 替换为你的HTTP触发器地址
notify_url: 'https://你的环境ID.service.tcloudbase.com/payCallback',
```

然后重新上传 `order` 云函数。

## 步骤4：在微信支付商户平台配置回调地址

1. 登录微信支付商户平台：https://pay.weixin.qq.com
2. 进入"产品中心" → "开发配置"
3. 找到"支付回调URL"配置项
4. 填入你的HTTP触发器地址：`https://你的环境ID.service.tcloudbase.com/payCallback`
5. 保存配置

## 回调云函数功能说明

`payCallback` 云函数会：
1. 接收微信支付服务器发送的支付结果通知
2. 验证签名确保数据来自微信
3. 解析XML格式的支付结果
4. 更新订单状态为"制作中"
5. 记录微信支付单号和支付时间
6. 返回成功响应给微信服务器

## 支付流程（完整版）

```
用户提交订单
    ↓
创建订单（状态：待支付）
    ↓
调用统一下单接口
    ↓
调起微信支付
    ↓
用户完成支付
    ↓
微信服务器发送回调通知 → payCallback云函数
    ↓                        ↓
前端收到支付成功          验证签名
    ↓                        ↓
前端更新订单状态          更新订单状态
    ↓                        ↓
清空购物车                记录支付信息
    ↓
跳转到订单页面
```

## 注意事项

1. **双重保障**：前端和回调都会更新订单状态，确保支付成功后订单一定会更新

2. **幂等性**：即使回调被多次调用，订单状态也只会更新一次

3. **安全性**：回调函数会验证微信签名，防止恶意请求

4. **日志记录**：回调函数会记录详细日志，便于排查问题

## 测试回调功能

部署完成后，可以通过以下方式测试：

1. **真实支付测试**：
   - 提交订单并完成支付
   - 查看云函数日志（云开发控制台 → 云函数 → payCallback → 日志）
   - 确认是否收到回调并成功更新订单

2. **模拟回调测试**：
   - 使用Postman等工具
   - 向HTTP触发器地址发送POST请求
   - 请求体为微信支付回调的XML格式数据

## 常见问题

### Q1: 找不到HTTP触发器地址
- 确认 `config.json` 文件配置正确
- 重新上传云函数
- 在云开发控制台查看触发器配置

### Q2: 回调没有被调用
- 检查商户平台是否配置了回调地址
- 查看云函数日志是否有请求记录
- 确认支付是否真的成功

### Q3: 签名验证失败
- 检查商户密钥是否正确
- 确认微信返回的数据格式正确
