# 数据库导入说明

## 导入步骤

### 1. 打开云开发控制台
- 在微信开发者工具中点击顶部"云开发"按钮
- 进入"数据库"标签

### 2. 创建集合
点击"+"号，依次创建以下集合：
- `users`
- `dishes`
- `orders`
- `order_items`

### 3. 导入数据

#### 方法一：使用导入功能（推荐）

1. **导入 users 集合**
   - 点击 `users` 集合
   - 点击"导入"按钮
   - 选择 `database/users.json` 文件
   - 点击确定

2. **导入 dishes 集合**
   - 点击 `dishes` 集合
   - 点击"导入"按钮
   - 选择 `database/dishes.json` 文件
   - 点击确定

3. **orders 和 order_items 集合**
   - 这两个集合暂时不需要导入数据
   - 用户下单后会自动生成数据

#### 方法二：手动添加（如果导入功能不可用）

1. 点击集合名称
2. 点击"添加记录"按钮
3. 切换到"JSON"模式
4. 复制 JSON 文件中的单条记录内容
5. 粘贴并保存
6. 重复以上步骤添加其他记录

## 数据说明

### users.json
包含2个测试账号：
- **管理员账号**：phone=admin, password=admin123
- **普通用户**：phone=13800138000, password=123456

### dishes.json
包含16道菜品，分为4个分类：
- **热菜**：6道（宫保鸡丁、鱼香肉丝等）
- **凉菜**：4道（酸辣土豆丝、拍黄瓜等）
- **汤类**：3道（西红柿鸡蛋汤、紫菜蛋花汤等）
- **主食**：3道（米饭、馒头、炒饭）

### orders.json 和 order_items.json
初始为空，用户下单后自动生成

## 注意事项

1. **导入前确保集合已创建**
2. **JSON 格式必须正确**（已经为你准备好了）
3. **导入后检查数据**：点击集合查看是否导入成功
4. **密码是明文存储**：实际项目中应该加密，这里为了测试方便使用明文

## 导入后的验证

导入完成后，检查：
- `users` 集合应该有 2 条记录
- `dishes` 集合应该有 16 条记录
- `orders` 和 `order_items` 集合为空（正常）

验证成功后，就可以继续下一步上传云函数了。
